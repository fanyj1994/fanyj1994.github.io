一个网页前端部分的介绍

2017-06-04
技术

我目前从事的职业是 Web 前端开发，很多人，尤其是文科背景的同学，听了之后，都说不明觉厉，似乎很高大上的样子。在现在这个人人可以学编程的年代，说实在，我不认为我们应该为学科设置什么不能触及的壁垒，就我个人而言，终生学习是我必须要践行的一个理念，所以，今天就来向大家科普一下前端开发是做什么的。

要说前端开发，先从开发或者编程，又或者写代码说起。我们通常所说的开发，就是使用一些特定的编程语言，写出计算机能够理解的代码，来告诉计算机该做些什么。事实上，代码也不是计算机直接能理解的，计算机只能理解 1 和 0 这两个数字，也就是我们所讲的二进制，开发者写出来的代码都要经过编译器编译成计算机能懂的二进制文件，一堆代码实现特定的功能组合，这就是我们常说的软件。

而软件开发，又因为操作系统以及平台的不同，所采用的开发模式以及编程语言也将会不同，我们所熟悉的 C 语言，在实际的生产环境下是用不到的，只是因为操作系统就是用它写成的，所以它才广为传颂，它就是一种最底层的编程语言。也就是说，我们最常用的 Windows 、Linux 以及 Mac OS，包括搭载在智能手机上的 Android 以及 IOS 等操作系统，都是使用 C 语言编写的。可以这样说：操作系统也是一款软件。一台没有装任何操作系统的电脑，是不具备任何功能的。我们真正用到的软件，例如 QQ ，或者 PS 等行业软件，或者各种管理系统，都是依托于操作系统运行的。由于操作系统的不同，开发这些软件所采用的编程语言也不同，运行在 Windows 上和 Mac 上的 QQ，其实是两个完全不同的语言编写的。

在操作系统之下，又有一些运行在特定容器内的应用软件，最常见的就是运行在浏览器内的网站，不管是打开浏览器逛淘宝还是看电影，其实都是在使用一些运行在浏览器内的软件，只不过由于浏览器的原因，这些软件不需要下载和安装。一个跑在互联网上的网站，一般意义上，都可以分为前端和后端，或者叫前台或者后台。所谓前端，就是我们浏览网页时候遇到的用户界面，用于与用户交互或者信息呈现。所谓后端，则是用于存放网络资源的服务器与数据库（大型网站会更复杂）。当我们访问一个网站，也就是输入一个网址，会由浏览器发送一个网络请求（一般是 HTTP 请求）到服务器，服务器接到请求后，会在自己的服务器或者数据库上面查找所请求的资源，如果能找到，就会将资源扒拉出来，通过网络扔给浏览器，浏览器会根据界面的设计将资源呈现出来。要实现这些功能，也需要特定的编程语言，但是，这样一个网站功能的实现，不是一门编程语言就可以实现的，而实现前端页面界面以及界面内的交互逻辑的实现（例如点击超链接怎么响应，鼠标点击图片图片放大以及网页上很炫的效果）等工作的，就是前端开发工程师，而当用户经由用户界面的操作，向服务器发送网络请求之后，服务器该怎么响应，例如我在百度输入一些搜索指令，服务器该如何从互联网上的一大堆资源中准确地找到你所搜索的信息，然后返回给前端的工作，则是后端开发者的任务。

下面重点介绍前端开发，前面说了，前端开发负责的是网页界面（UI）的呈现，也就是一个网站，该怎么排版布局，色彩搭配，中间要加一些什么动态效果，这些东西最初会有专门的设计师设计出来，画成图，然后扔给前端工程师，前端工程师就需要写代码，将这些东西呈现出来，要求是还原设计稿。要实现这些，前端开发主要使用到三项技术：HTML, CSS 以及 JavaScript，其中 HTML 是一个网页的骨骼，在一个网页中，我们所看到的所有元素，在网页代码中都会有一个标签来代表它，这些标签都很语义化，例如要放一张图片，则需要一个 <img> 标签，一个标题，则需要一个 <h1>（或者<h2345>），很好理解，h 就是 head 的意思，用 <p> 标签表示段落( paragraph )。我们用电脑打开一个网站，例如腾讯首页，它是这幅样子：

现在，借助 HTML + CSS，我们已经可以写出很好看的静态网页页面了，但是，作为一个网页前端工程师，只会写这样的东西是找不到工作的，前端工程师更重要的是，需要掌握第三大剑客 JavaScript 语言。可以说，严格地讲，HTML 和 CSS 不能算作编程语言，因为它们没有变量、函数等最基本的编程语言特性，那么，JavaScript（下简称 JS） 语言可就是一门真正的而且越来越流行，受重视的编程语言了。JS 负责的是一个页面的交互逻辑的实现，例如我们常见到的网页上的动态效果，自动切换的轮播图，标签页，返回顶部按钮，什么时候元素隐藏，什么时候出现，都可以用 JS 搞定，还是刚才的例子，简单地用 JS 实现一个鼠标放在图片上的时候，图片的边框变色的效果。

至此，我们已经了解了前端开发的主要三个内容。可以说，学好这三样子，我们已经可以作出各种各样的网页了。但是，在实际的工作业务需求中，事情会比这复杂的多， HTML 我们需要考虑到语义良好，元素简洁， CSS 也经常借助一些预处理语言(如 Sass、Less等)以及一些高度包装的框架（Bootstrap等）来更高效完成工作，至于 JS，则更是有许多日新月异的框架产生，可以这么说，基本每隔一两年，前端开发者所使用的技术栈都会革新一大部分，所以，要做一名好的开发者，必须要保持不断的学习力度，才能够做一名合格的 Web 开发程序员。

以上就是前端开发的基本情况，总结一下，前端开发工程师不再是很久以前被大家看低的页面仔或者切图的，在互联网时代，随着 Web 开发的工程化，Web 前端页面的开发，已经成为一种 GUI 软件的实现，期间所需要的交互逻辑以及工程化的解决方案，变得越来越复杂，但是，这不正是一个开发者的挑战所在吗？不管是学了什么专业，或者从事什么行业，在这个时代，我认为不要固步自封，要不断学习新事物才是可取之道，这也是我推荐很多人去学习一些编程的原因，因为有的时候，只有你不把它当做一门谋生手段的时候，做这件事情的乐趣才能更多地体现出来。就像我，不管是文学与写作，还是哲学与绘画，我觉得都是一些让人乐趣无穷的活动，都值得我们认真学习，探讨。
